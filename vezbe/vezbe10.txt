Vežba 10: Blokirajuće i neblokirajuće
utičnice
Utičnice mogu izvršavati ulazno-izlazne operacije u blokirajućem i neblokirajućem režimu.
Razlika između ova dva režima je u tome što, kada god bi operacija nad blokirajućom
utičnicom zaustavljala izvršenja glavnog programa, neblokirajuća utičnica ne bi dozvolila
operaciju već bi vratila specijalnu "would block" poruku greške.
Pokušajmo dočarati blokirajuće i neblokirajuće režime kroz situacije iz svakodnevnog života.

Ista operacija pozvana u različitim trenucima može jednom blokirati izvršenje, a drugi put ne.
U nastavku su data četiri tipa operacija i uslovi pod kojima će te operacije blokirati izvršenje:
 Operacije čitanja (Receive, ReceiveFrom) – primalac poruke će biti blokiran sve dok
poruka koju očekuje ne pristigne sa mreže. Ako nema dostupnih podataka, niti je
postavljen timeout, operacija će čekati na dolazne podatke.
 Operacije pisanja (Send, SendTo) – pošiljalac poruke će biti blokiran ako nema
dovoljno slobodnog mesta u baferu za slanje poruke. Operacija se nastavlja kada se
oslobodi prostor u baferu.
 Operacija prijema konekcija (Accept) – server će biti blokiran do trenutka kada stigne
novi zahtev za uspostavu konekcije. Ako nema dolaznih zahteva, operacija će ostati
blokirana dok se ne pojavi klijent koji traži povezivanje.
 Operacija iniciranja konekcije (Connect) – klijent će biti blokiran do trenutka uspešnog
uspostavljanja TCP konekcije (protokol rukovanja). Ako veza ne može da se uspostavi
(npr. server nije dostupan), operacija će ostati blokirana dok ne istekne timeout ili dok
se ne javi greška u konekciji.

Vežba 10: Blokirajuće i neblokirajuće utičnice

1. Blokirajući soketi
Blokirajuće utičnice čekaju izvršenje operacije bez obzira na to da li će se ona izvršiti
momentalno, posle nekog vremena ili se možda nikad neće izvršiti. Problem je, pre svega, u
tome što je nit onemogućena da za vreme čekanja obavlja još neke zadatke.

program: pošalji mi ove podatke
socket: okej, ali znaj da operacija može potrajati
neko vreme
...
...
...
socket: poslao
program: hvala, nastavljam sa radom

Slika 1. Blokirajući soket
Bitno je napomenuti da se za vreme čekanja izvršavanja ne troši dragoceno procesorsko
vreme. Iako je nit programa blokirana, to ne sprečava druge niti da koriste procesor.
Sve utičnice koje su korišćene na prethodnim vežbama bile su u blokirajućem režimu koji
predstavlja podrazumevani (eng. default) režim rada utičnice. Nasuprot njima su
neblokirajuće utičnice, čija implementacija je zahtevnija, ali omogućava znatno bolje
performanse komunikacije.

2. Neblokirajuće utičnice
Funkcije koje se pozivaju nad utičnicom koja je postavljena u neblokirajućem režimu nikada
neće blokirati izvršavanje programa, već će povratak iz funkcije uslediti odmah nakon poziva.
Neblokirajuće operacije nad utičnicom će se izvršiti, dok će blokirajuće generisati grešku koja
će ukazati da nije bilo dovoljno vremena da se operacija izvrši.
U .NET.Sockets biblioteci, utičnica se može postaviti u neblokirajući režim postavljanjem
svojstva Blocking na false. Ova operacija se može primeniti na sve tipove utičnica, bez obzira
na stanje u kojem se nalaze. Takođe, funkcije koje omogućavaju asinkronu obradu podataka
mogu implicitno postaviti režim soketa na neblokirajući.

Deklaracija svojstva Blocking, njegov detaljan opis i primer su dati u nastavku:
socket.Blocking = false;

U nastavku je dat primer u kome se koristi neblokirajuci režim rada TCP utičnice.
// Postavljanje utičnice u neblokirajući režim
Socket clientSocket = new Socket(AddressFamily.InterNetwork, SocketType.Stream,
ProtocolType.Tcp);
clientSocket.Blocking = false;
// Poziv operacije nad utičnicom
try
{
int bytesReceived = clientSocket.Receive(dataBuffer);
// Provera da li je operacija uspešno izvršena
if (bytesReceived > 0)
{
// Neblokirajuća operacija uspešno izvršena
Console.WriteLine("Podaci primljeni: " + Encoding.UTF8.GetString(dataBuffer, 0,
bytesReceived));
}
}
catch (SocketException ex)
{
if (ex.SocketErrorCode == SocketError.WouldBlock)
{
// U pitanju je blokirajuća operacija koja zbog režima utičnice neće biti izvršena
Console.WriteLine("Operacija bi blokirala, nastavljam dalje...");
}
else
{
// Desila se neka druga greška prilikom poziva operacije
Console.WriteLine("Socket error: " + ex.SocketErrorCode);
}
}

Na početku je potrebno postaviti utičnicu u neblokirajući režim postavljanjem svojstva Blocking
na false. Nakon toga, poziva se operacija Receive nad utičnicom. Ako operacija ne može biti
izvršena zbog neblokirajućeg režima, baca se izuzetak SocketException, a kod greške
SocketError.WouldBlock ukazuje da bi operacija blokirala, ali zbog režima utičnice nije bila
dozvoljena. Ako je operacija uspešno izvršena, podaci se normalno primaju i obrađuju.

Polling model
Polling model podrazumeva ponavljanje poziva operacije nad neblokirajućim soketom, dokle
god se operacija ne izvrši uspešno.
program: pošalji mi ove podatke
socket: ne mogu, operacija zahteva blokiranje
...
program: pošalji mi ove podatke
socket: ne mogu, operacija zahteva blokiranje
...
program: pošalji mi ove podatke
socket: ne mogu, operacija zahteva blokiranje
...
program: pošalji mi ove podatke
socket: poslao

Slika 2 – Polling model

U odnosu na blokirajući režim, ovaj model neblokiranja pruža određenu vrstu kontrole u
okviru petlje koja omogućava da se za vreme čekanja izvrše još neke druge operacije ili da
se prekine čekanje kada se, na primer, nekoj promenljivoj promeni vrednost. S druge
strane, za vreme čekanja se troši dragoceno procesorsko vreme i glavni program je za
vreme čekanja blokiran.
U .NET Socket biblioteci, polling model se implementira pomoću metode Poll. Ova metoda
omogućava programu da proveri da li je utičnica spremna za čitanje, pisanje, ili da li je
došlo do izuzetka.
Funkcija Poll vraća vrednost true ako utičnica zadovoljava određeni uslov (čitanje, pisanje ili
greška), ili false ako nije spremna.

Deklaracija metode Poll
public bool Poll(int microSeconds, SelectMode mode);

Parametar
Opis
microSeconds [int] Broj mikrosekundi koliko će utičnica čekati da zadovolji određeni uslov (0 za
momentalni povratak).
mode [SelectMode] Vrednost SelectMode, koja može biti SelectRead, SelectWrite ili SelectError.

SelectRead: Proverava da li je utičnica spremna za čitanje.
SelectWrite: Proverava da li je utičnica spremna za pisanje.
SelectError: Proverava da li je došlo do greške.
Povratna vrednost
Opis
bool Metoda vraća true ako je utičnica spremna za operaciju u navedenom režimu, ili
false ako nije.

Primer:
// Definisanje i povezivanje soketa
Socket listenSocket = new Socket(AddressFamily.InterNetwork, SocketType.Stream,
ProtocolType.Tcp);
listenSocket.Bind(new IPEndPoint(IPAddress.Any, 51000));
listenSocket.Listen(10);
Socket acceptedSocket = listenSocket.Accept();
while (true)
{
// Polling model - čekanje da utičnica bude spremna za prijem
if (acceptedSocket.Poll(1000 * 1000, SelectMode.SelectRead)) // 1000 ms
{
byte[] dataBuffer = new byte[256];
int bytesRead = acceptedSocket.Receive(dataBuffer);
string message = Encoding.UTF8.GetString(dataBuffer, 0, bytesRead);
Console.WriteLine("Primljena poruka: " + message);
}
else
{
Console.WriteLine("Još uvek čekam...");
}
}
