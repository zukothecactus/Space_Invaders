U prethodnoj vežbi upoznali smo neblokirajući režim rada utičnica, koji omogućava da se
izbegne blokiranje programa ako operacija nad utičnicom nije spremna da se izvrši. Međutim,
teško je bilo odrediti kada je pravi trenutak za novi pokušaj. Zbog toga se periodično iznova
pokušava izvršiti operacija trošeći procesorsko vreme.

Metoda Socket.Select omogućava praćenje stanja čitanja, pisanja i grešaka na više utičnica
odjednom, omogućavajući pravovremenu reakciju na događaje od interesa. Na ovaj način se
izbegava potreba za periodičnim proverama stanja objekata od interesa. Na primer, kod TCP
servera može se istovremeno čekati na prispeće poruka od konektovanih klijenata i na nove
zahteve za uspostavom konekcije.

Select model

Funkcija Socket.Select omogućava praćenje utičnica za čitanje, pisanje i greške, koristeći tri
različite kolekcije utičnica.
Metoda Socket.Select ima sledeću deklaraciju:
public static void Select(IList checkRead, IList checkWrite, IList checkError, int
microSeconds);

Funkcija:

Opis:

Select

Omogućava praćenje utičnica za čitanje, pisanje
i greške, koristeći tri različite kolekcije utičnica.

Parametri:

Opis:
checkRead[IList] Lista utičnica koje se proveravaju za mogućnost čitanja
(Socket.Receive, Socket.Accept).
checkWrite[IList] Lista utičnica koje se proveravaju za mogućnost pisanja
(Socket.Send, Socket.Connect).
checkError[IList] Lista utičnica koje se proveravaju za greške.
microSeconds[int] Maksimalno vreme čekanja u mikrosekundama. Ako se
postavi na 0, funkcija se ne blokira i odmah vraća
rezultat. Ako se postavi na -1, funkcija će čekati
neograničeno dugo dok se neki događaj ne dogodi.

Može se primetiti da postoje tri liste: jedna za praćenje čitanja (checkRead), jedna za praćenje
pisanja (checkWrite) i jedna za praćenje grešaka (checkError). Ovi ulazno-izlazni parametri
predstavljaju skup utičnica za koje je potrebno pratiti određeni status. Nakon registrovanja
događaja od interesa, metoda Socket.Select će u ovim listama ostaviti samo one utičnice na
kojima su događaji zabeleženi.
Na ovaj način, moguće je saznati koji se događaj desio tako što ćemo proveriti da li utičnica i
dalje postoji u odgovarajućoj listi nakon izvršenja metode Select. Na primer, ako je potrebno
proveriti mogućnost čitanja nad utičnicom, dovoljno je dodati utičnicu u listu checkRead pre
poziva metode Select. Nakon poziva, ako se utičnica i dalje nalazi u ovoj listi, možemo zaključiti
da je spremna za operaciju čitanja.
Ovo omogućava da se na osnovu rezultata metode Socket.Select odmah započne
odgovarajuća operacija. Na primer, za operaciju čitanja može se odmah pozvati metoda
Receive, ako je utičnica označena kao spremna za čitanje. Na taj način se izbegava nepotrebno
blokiranje i efikasno koristi procesorsko vreme.

Alogritam za multipleksiranje operacija koje je zasnovano na
događajima
1. Postavljanje soketa u neblokirajući režim
listenSocket.Blocking = false; // Postavi soket u neblokirajući režim

2. Inicijalizacija kolekcija za praćenje
Koristi se generička lista (List<Socket>) za praćenje utičnica.
List<Socket> readSockets = new List<Socket>();
readSockets.Add(clientSocket); // Dodajemo klijentsku utičnicu

3. Poziv metode Select za praćenje događaja
Socket.Select se poziva sa željenim listama i vremenom čekanja.
Socket.Select(readSockets, null, null, 1000000); // Čekamo do 1 sekunde

4. Provera rezultata poziva Select metode
Nakon poziva metode Select, ako je utičnica spremna za čitanje, lista readSockets će i
dalje sadržati tu utičnicu.
if (readSockets.Count > 0)
{
// Spremno za čitanje
byte[] buffer = new byte[256];
int bytesRead = clientSocket.Receive(buffer);
string message = Encoding.UTF8.GetString(buffer, 0, bytesRead);
Console.WriteLine("Primljena poruka: " + message);
}
else
{
Console.WriteLine("Vreme čekanja je isteklo, nema događaja.");
}

5. Čišćenje kolekcija za novu iteraciju
Pre nove iteracije, kolekcije se prazne ili se resetuju za ponovno praćenje.
readSockets.Clear(); // Očisti listu za sledeću iteraciju
readSockets.Add(clientSocket); // Ponovo dodaj klijentsku utičnicu

Kompletni primer u .NET-u:
// Postavljanje utičnice u neblokirajući režim
Socket listenSocket = new Socket(AddressFamily.InterNetwork, SocketType.Stream,
ProtocolType.Tcp);
listenSocket.Bind(new IPEndPoint(IPAddress.Any, 51000));
listenSocket.Listen(10);
Socket clientSocket = listenSocket.Accept();
clientSocket.Blocking = false; // Neblokirajući režim
while (true)
{
// Inicijalizacija kolekcije za praćenje događaja
List<Socket> readSockets = new List<Socket> { clientSocket };
// Poziv metode Select sa timeoutom od 1 sekunde
Socket.Select(readSockets, null, null, 1000000);
// Provera rezultata Select metode
if (readSockets.Count > 0)
{
byte[] buffer = new byte[256];
int bytesRead = clientSocket.Receive(buffer);
string message = Encoding.UTF8.GetString(buffer, 0, bytesRead);
Console.WriteLine("Primljena poruka: " + message);
}
else
{
Console.WriteLine("Vreme čekanja je isteklo, nema događaja.");
}

}

// Čišćenje seta za novu iteraciju
readSockets.Clear();

Ovaj primer koristi metodologiju zasnovanu na događajima za praćenje TCP utičnice, uz
izbegavanje neefikasnog trošenja procesorskog vremena i omogućavanje paralelne obrade
drugih zadataka dok čekamo na događaj.

Slanje složenih tipova podataka
Za slanje složenih tipova podataka (i kolekcija: listi, nizova...), kako bi se oni pretvorili u niz
bajtova koji odgovara parametrima funkcija za slanje, odnosno prevođenje primljenih podataka
u originalne tipove, koristiće se klase MemoryStream i BinaryFormatter.
Klasa MemoryStream predstavlja način da se podaci čuvaju direktno u memoriji, umesto u
datoteci ili kao resurs, što može korisno kada se manipulacija nad podacima radi u memoriji i ne
koristi ulazno/izlazne operacije. Ona implementira IDisposable interfejs koji nudi mogućnost
oslobađanja zauzetih resursa, iako sam MemoryStream nema resurse koje je potrebno
osloboditi.

Klasa BinaryFormatter omogućava serijalizaciju i deserijalizaciju objekata:
 Serijalizacija predstavlja proces konverzije objekata u binarni format. Objekat klase koji
se serijalizuje mora imati atribut [Serializable] nad svojom klasom.
using (MemoryStream ms = new MemoryStream())
{
BinaryFormatter bf = new BinaryFormatter();
bf.Serializer(ms, objekat);
dataBuffer = ms.ToArray();
}

 Deserijalizacija predstavlja proces konverzije binarnog formata bajtova u originalni
objekat.

using (MemoryStream ms = new MemoryStream(dataBuffer))
{
BinaryFormatter bf = new BinaryFormatter();
Klasa objekat = (Klasa)bf.Deserializer(ms);
}

Liste i nizovi se mogu serijalizovati sa elementima koji su opšti tipovi, ali za složenije, se mora
navoditi atribut [Serializable] (strukture i klase).

Programsko pokretanje više klijenata

Za programsko pokretanje više klijenata, koristi se C# klasa Process, koja pripada biblioteci
System.Diagnostics i omogućava pokretanje i zaustavljanje sistemskih procesa. Kao svojstvo
naziva procesa koji se pokreće se navodi konkretan klijent. Za porketanje samog procesa, nad
njim se poziva metoda Start.
static void PokreniKlijente(int brojKlijenata)
{
for (int i = 0; i < brojKlijenata; i++)
{
// Putanja do izvršnog fajla klijenta (potrebno je kompajlirati ga)
string clientPath = @"C:\..\UDPClient.exe";
Process klijentProces = new Process(); // Stvaranje novog procesa
klijentProces.StartInfo.FileName = clientPath; //Zadavanje putanje za pokretanje
klijentProces.StartInfo.Arguments = $"{i + 1}"; // Argument - broj klijenta
klijentProces.Start(); // Pokretanje klijenta
Console.WriteLine($"Pokrenut klijent #{i + 1}");
}
}
